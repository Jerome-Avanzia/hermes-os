# Contributing to Hermes OS

Thank you for contributing to Hermes OS.

## Core Principles

- Documentation First
- Git First
- Security by Design
- AI Assists – Humans Approve

## Workflow

1. Plan
2. Design
3. Generate
4. Technical Review
5. Security Review
6. Owner Approval
7. Commit
8. Push

## Commit Messages

Use Conventional Commits when possible.

Examples:

- docs(governance): update operating model
- feat(agent): add agent registry
- fix(workflow): correct metadata template
- chore(repo): add gitignore

## Controlled Documents

All controlled documents must follow:

- standards/document-metadata.md
- templates/controlled-document.md

## Reviews

Every governance document requires:

- Technical review
- Security review
- Owner approval

## Branch Strategy

Current policy:

- main is the protected baseline.
- Small, atomic commits.
- One logical change per commit.

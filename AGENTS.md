## Purpose

This repository is the architecture-first source of truth for Hermes OS. It defines the operating concepts, systems and decisions that support AI-enabled businesses in the AVANZIA ecosystem.

## Working Rules

Always:

- Work only inside the current repository.
- Review relevant documentation before editing.
- Preserve approved architecture and specifications.
- Document significant decisions before implementation.
- Keep changes small, clear and version-controlled.
- Show a diff and wait for review when requested.

Never:

- Invent architecture decisions.
- Implement code unless explicitly requested.
- Modify unrelated files.
- Replace approved documentation with generic content.

## Documentation Standards

Use concise Markdown with a title, version, status, owner, maintainer, purpose and document status where applicable. Prefer clear headings and short paragraphs.

## Decision Making

Follow approved specifications exactly. If documentation conflicts or intent is unclear, request clarification rather than guessing.

## Definition of Done

A task is complete when the requested documentation is accurate, formatted consistently, limited to the requested scope and ready for review.
